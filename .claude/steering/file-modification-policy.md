# 文件修改策略

## 核心规则

在执行以下操作前，**必须**先向用户说明计划并等待明确确认：

- 修改任何现有文件
- 创建新文件
- 删除文件
- 重命名文件

## 工作流程

1. **分析阶段**: 可以自由使用 Read、Grep、Glob 等工具分析代码
2. **计划阶段**: 说明将要进行的修改内容和原因
3. **等待确认**: 明确询问用户 "是否确认执行？"
4. **执行阶段**: 仅在收到用户明确同意后才使用 Write、Edit 等修改工具

## 示例

❌ 错误做法：
```
我发现了问题，现在修复它...
[直接调用 Edit 工具]
```

✅ 正确做法：
```
我发现了问题，建议修改 src/main.rs 第 42 行...
计划的修改内容：[说明具体改动]

是否确认执行这些修改？
[等待用户回复]
```

## 例外情况

无例外。所有文件操作都需要确认。
